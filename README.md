# 🎧 Patrick Speak — YouTube 英文聽力學習套件

把任何一支有講英文的 YouTube 影片，**全自動**變成一套分段的英文聽力教材，支援單句重複播放與聽寫練習。全程在你自己的電腦上運算，不需任何雲端 API、不上傳資料、不碰瀏覽器 cookies。

> 靈感來自市面上以 TED 內容做的英文學習 App，但這套可以用**任何** YouTube 影片，而且完全離線、免費、可自由擴充。

## 功能特色

- **貼上連結即可**：輸入 YouTube 網址，自動下載音訊、轉錄、切句、翻譯。
- **智慧切段**：以「完整句子」為基準，每接近 2 分鐘切成一段教材，長度適中好消化。
- **兩種學習模式**：
  - 📖 **教材播放模式** — 單句重複、上一句／下一句，可即時切換顯示英文字幕、繁體中文翻譯。
  - ✍️ **聽寫練習模式** — 播放單句後自動暫停 10 秒供你打字，逾時自動重播；可隨時查看英文與繁中參考。
- **長片友善**：預設只處理前 6 分鐘（快速可用），需要整支時按「全部下載」補完；已翻譯的句子不會重翻。
- **影片庫側欄**：處理過的影片列在右側，點一下從快取秒載，免重新處理。
- **播放速度**：0.75x / 1.0x / 1.25x，初學放慢、進階加速。
- **我的最愛**：點 ★ 收藏難句，側欄集中複習、可一鍵跳回該句。
- **亮/暗主題**：右上角一鍵切換。
- **不依賴 YouTube 字幕**：用本地 Whisper 直接聽音訊轉文字，所以即使影片**沒有提供英文字幕也能用**。
- **完全本機、隱私安全**：音訊、轉錄、翻譯都在你電腦上完成，不需登入 YouTube、不需任何 API 金鑰。
- **自動快取**：同一支影片第二次開啟即秒開。

## 系統架構

```
                    ┌──────────────────────────────────────────────┐
   YouTube 連結 ──▶ │  FastAPI 後端 (backend/main.py)               │
                    │                                                │
                    │  1. yt-dlp ─────▶ 下載音訊 (android 客戶端)    │
                    │  2. ffmpeg ─────▶ 轉 16kHz 單聲道 WAV          │
                    │  3. whisper.cpp ▶ 語音轉文字（帶時間戳）       │
                    │  4. 切句邏輯 ───▶ 重組成完整句子               │
                    │  5. 切段邏輯 ───▶ 每 ~2 分鐘一段               │
                    │  6. Ollama ─────▶ 逐句英翻繁中                 │
                    │  7. ffmpeg ─────▶ 依段落切出音訊片段           │
                    └───────────────────────┬──────────────────────┘
                                            │ JSON
                    ┌───────────────────────▼──────────────────────┐
                    │  前端 (原生 HTML/CSS/JS，由後端直接 serve)     │
                    │  教材播放模式 ／ 聽寫練習模式                  │
                    └──────────────────────────────────────────────┘
```

| 環節 | 使用工具 | 說明 |
|------|----------|------|
| 下載音訊 | `yt-dlp` + `ffmpeg` | 以 android 客戶端抓取，繞過登入驗證，不需 cookies |
| 語音轉文字 | `whisper.cpp` (`whisper-cli`) | 本地轉錄成帶時間戳的英文，**不依賴 YouTube 字幕** |
| 智慧切句 | 自製邏輯 | 把 Whisper 片段重組成完整句子，時間戳按字數比例估算 |
| 切段教材 | 自製邏輯 | 以句子為基準，每接近 2 分鐘切一段 |
| 英翻繁中 | Ollama (`translategemma` 等) | 逐句翻譯確保中英一對一對齊 |
| 前端介面 | 原生 HTML/CSS/JS | 無需建置流程，後端直接提供 |

技術棧細節見 [docs/TECHNICAL.md](docs/TECHNICAL.md)。

## 設備建議

本專案是「**本地 AI 運算**」應用，效能主要取決於記憶體與晶片：

| 等級 | 建議規格 | 體驗 |
|------|----------|------|
| 最低 | Apple Silicon (M1/M2/M3/M4)，**16GB** 統一記憶體 | 可跑。建議翻譯用較小模型（如 `gemma3:12b` 的 q4 量化，約 7GB）。同時別再開其他大型模型。 |
| 建議 | Apple Silicon，**24GB+** | 順暢。可常駐翻譯模型 + Whisper，瀏覽器、系統都不卡。 |
| 充裕 | Apple Silicon，**32GB 以上**（本專案實測於 M 系列 68GB） | 隨意，甚至可換 `medium.en` 大模型提升轉錄準確度。 |

**重要經驗**：本地模型吃的是「統一記憶體」。如果你**同時**開著其他大型模型（例如另一個 30B 的 server），會互相搶記憶體頻寬，導致翻譯速度從每秒數十 token 掉到每秒 1 個 token（慢數十倍）。跑本專案時請確認沒有其他大型模型在背景佔用記憶體。

> Windows / Linux 也能跑（whisper.cpp 與 Ollama 皆跨平台），但本指南以 macOS + Homebrew 為主。

## 安裝指南

### 快速安裝（推薦，一鍵搞定）

只需要先手動裝好兩個「App 等級」的東西，其他都交給腳本：

1. **Homebrew**（若尚未安裝）：到 [brew.sh](https://brew.sh) 依網站指示安裝。
2. **Ollama**：到 [ollama.com/download](https://ollama.com/download) 下載安裝，並**打開它一次**（它會在背景常駐執行，負責英翻中）。

接著在終端機切到本專案資料夾，執行：

```bash
./setup.sh
```

這個腳本會自動：安裝 yt-dlp / ffmpeg / whisper-cpp、下載翻譯模型、建立 Python 環境、下載 Whisper 語音辨識模型（約 466MB，需要一點時間）。全部跑完看到「安裝完成！」就好了。

之後每次要用，執行：

```bash
./start.sh
```

會自動開啟瀏覽器到 `http://localhost:8000`。要結束就回到終端機視窗按 `Ctrl + C`。

> 想手動照步驟安裝、或了解每個工具的用途，請見下方「手動安裝步驟」。

---

### 手動安裝步驟

### 1. 安裝系統工具（yt-dlp、ffmpeg、whisper.cpp）

先安裝 [Homebrew](https://brew.sh)（若尚未安裝），然後：

```bash
brew install yt-dlp ffmpeg whisper-cpp
```

安裝後確認 whisper 指令名稱（不同版本可能是 `whisper-cli`）：

```bash
which whisper-cli
```

### 2. 安裝並設定 Ollama（翻譯）

Ollama 是本地大型語言模型的執行平台，負責英翻中。

1. 到 [ollama.com/download](https://ollama.com/download) 下載安裝（或 `brew install ollama`）。
2. 啟動 Ollama（安裝後通常會自動在背景執行，於 `http://localhost:11434`）。
3. 下載一個翻譯模型，擇一即可：

```bash
# 選項 A：translategemma — 專為翻譯訓練，速度快、品質好（本專案預設）
ollama pull translategemma

# 選項 B：gemma3:12b — Google 通用模型，中文翻譯也不錯，16GB 機器友善
ollama pull gemma3:12b

# 選項 C：qwen 系列 — 中文表現強，但部分使用者對中國模型有資安顧慮（即使本地執行）
ollama pull qwen2.5:7b
```

> 模型選擇沒有標準答案：要中文最自然，Qwen 系列通常最強；在意「非中國模型」可選 Gemma 系列；要專注翻譯任務則 translategemma。詳細權衡見 [docs/TECHNICAL.md](docs/TECHNICAL.md)。

確認模型已下載：

```bash
ollama list
```

### 3. 下載 Whisper 英文模型

Whisper 的語音辨識模型需單獨下載一次（`small.en`，約 466MB，英文專用、準確度與速度平衡好）：

```bash
cd Patrick-Speak
mkdir -p models
curl -L -o models/ggml-small.en.bin \
  https://huggingface.co/ggerganov/whisper.cpp/resolve/main/ggml-small.en.bin
```

想要更高準確度可改下載 `ggml-medium.en.bin`（約 1.5GB，較慢），並在 `.env` 調整路徑。

### 4. 安裝專案

```bash
cd Patrick-Speak
python3 -m venv .venv
.venv/bin/pip install -r requirements.txt
```

## 操作說明

1. 在輸入框貼上 YouTube 連結，按「**開始處理**」。
2. 預設只處理**前 6 分鐘**（較快可用）；完成後會**快取**，同影片再開即秒出。
3. 上方會出現一排「**第 N 段**」分頁，點選切換不同段落教材。
4. 用兩顆按鈕切換 **教材播放模式** 與 **聽寫練習模式**。

**長片：全部下載**
影片超過 6 分鐘時，上方會出現「⬇ 全部下載」。按下才會處理整支；已翻譯的前 6 分鐘不會重翻。處理過的網址，「開始處理」會自動變灰（改網址才會重新啟用）。

**鍵盤快捷鍵：**

| 按鍵 | 功能 |
|------|------|
| `空白鍵` | 播放 / 暫停目前句 |
| `←` | 上一句 |
| `→` | 下一句 |
| `R` | 重播目前句 |

**字幕顯示**：兩種模式都有「英文字幕」「中文翻譯」兩顆開關，可獨立隨時開關。

**播放速度**：播放器下方可選 0.75x / 1.0x / 1.25x，偏好會記住。

**聽寫模式流程**：播放單句 → 自動暫停並倒數 10 秒 → 你在輸入框打字 → 按 Enter 或「送出答案」比對 → 若 10 秒內未操作則自動重播該句。

### 右側側欄

- **📚 我的影片庫**：列出處理過的影片，點一下從快取秒載。每項可 **▲▼ 排序**、**✕ 刪除**（刪除會清掉該影片快取、釋放硬碟空間）。
- **⭐ 我的最愛**：在句子旁點 **★** 收藏難句；側欄集中複習，可 **▲▼ 排序**、點一下**跳回**該句、**✕ 移除**。
- 影片庫與最愛**各自獨立捲動**，影片再多最愛也永遠看得到。
- 側欄頂端可切換 **🌙 暗色 / ☀️ 亮色** 主題。

> 影片庫順序、最愛、播放速度、主題都存在瀏覽器本機（localStorage）。

## 設定檔（可選）

複製 `.env.example` 成 `.env` 即可覆寫預設值：

```bash
cp .env.example .env
```

| 變數 | 預設 | 說明 |
|------|------|------|
| `OLLAMA_URL` | `http://localhost:11434` | Ollama 服務位址 |
| `OLLAMA_MODEL` | `translategemma:latest` | 翻譯用模型 |
| `WHISPER_BIN` | `whisper-cli` | whisper.cpp 執行檔名稱 |
| `WHISPER_MODEL` | `models/ggml-small.en.bin` | Whisper 模型路徑 |
| `PREVIEW_SECONDS` | `360` | 預設處理的秒數（前 6 分鐘）；按「全部下載」會處理整支 |

## 分享給朋友

**不要直接把整個資料夾 zip 起來傳**——裡面的 `.venv`（Python 環境）、`audio_cache`（你的下載快取）、`models`（大型模型檔）都是本機專屬或體積龐大的內容，直接打包會超過 1GB，而且 `.venv` 在別人電腦上會直接壞掉。

改用內附的打包腳本：

```bash
./pack.sh
```

會產生一個乾淨的 `PatrickSpeak.zip`（幾十 KB），已自動排除上述內容。傳給朋友後，請他們：

1. 解壓縮
2. 開終端機切到解壓縮後的資料夾
3. 執行 `./setup.sh`（第一次安裝，含前置需求：Homebrew、Ollama）
4. 執行 `./start.sh`（之後每次啟動）

朋友只需要會**開終端機、貼指令**即可，不需要 npm、不需要懂 Python。

## 常見問題

**Q：出現「Sign in to confirm you're not a bot」？**
本專案已改用 yt-dlp 的 android 客戶端繞過此驗證，正常情況不會遇到。若仍遇到，可能是該影片有特殊限制（會員限定、地區封鎖），換一支公開影片即可。**不需要、也不建議**動用瀏覽器 cookies。

**Q：處理很慢？**
確認沒有其他大型模型佔用記憶體（見[設備建議](#設備建議)）。第一次處理本來就需數分鐘，之後快取秒開。

**Q：翻譯偶爾怪怪的？**
目前為逐句翻譯以確保對齊穩定，少數缺上下文的句子可能略有瑕疵。可換更強的翻譯模型，或見 [docs/ROADMAP.md](docs/ROADMAP.md) 的「上下文翻譯」改進方向。

**Q：想清空快取？**
刪除 `audio_cache/` 資料夾內容即可。
