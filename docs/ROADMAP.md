# 未來擴展功能 (Roadmap)

依「實作難度」與「對學習體驗的提升」整理。可挑你最想要的，當作下一輪 Claude Code 的練習題。

---

## ✅ 已完成

- **影片庫側欄**：已處理影片列在右側，點一下從快取秒載（後端 `_meta.json` + `/api/library`）。可 **▲▼ 排序**（順序存 localStorage）與 **✕ 真刪除**（後端 `DELETE /api/library/{id}`，連帶釋放硬碟空間）。
- **側欄版面**：影片庫與最愛各自獨立捲動，影片再多最愛也永遠看得到。最愛項目也可 **▲▼ 排序**。
- **預設 6 分鐘 + 全部下載**：長片預設只處理前 6 分鐘，按「全部下載」補完剩餘；翻譯快取讓補完不重翻已翻句子（`PREVIEW_SECONDS`、`_trans.json`）。
- **播放速度**：0.75x / 1.0x / 1.25x，偏好存 localStorage。
- **我的最愛**：點 ★ 收藏句子，側欄集中複習、可跳轉與刪除（localStorage）。
- **亮/暗主題切換**：右上角切換，偏好存 localStorage。
- **穩健性**：單句翻譯失敗不再拖垮整批、巨句保護、並行保險（同時只處理一支）、前端錯誤訊息友善化、前端檔案 no-cache（不再看到舊版畫面）。
- **喜歡的頻道（左側欄）**：輸入框旁 ★ 一鍵收藏當前影片的頻道；也可貼頻道網址加入；以頻道名稱顯示，點一下開啟 YouTube 頻道頁。
- **音樂模式 🎵**：貼 YouTube / YouTube Music 歌曲連結，用 lrclib 同步歌詞（LRC）取代 whisper（唱歌對 whisper 是硬傷），逐句時間軸精準；只收英文歌詞（非英文明確拒絕）；整首處理、支援逐句播放與歌詞聽寫。
  - 歌詞比對優先用 yt-dlp 的精確 `track`/`artist` 欄位（YouTube Music 常見）直接查詢，避免像單字標題（如 "Babe"）模糊搜尋配到完全不相干的同名歌曲；沒有精確欄位時才退回模糊標題搜尋，並以「時長最接近」挑版本。
  - 歌曲切段放寬為 10 分鐘一段（`MUSIC_SEGMENT_DURATION`），因為靠下載歌詞不需頻繁分段，多數歌曲一段就結束。
- **歌詞時間軸對齊**（兩層防護）：
  - **手動微調鈕（保底）**：音樂模式播放器有「歌詞對齊 ±0.5s」，任何來源的偏移都能當場聽著修正，按歌曲存 localStorage。
  - **VAD 挑候選（自動加分）**：用 Silero VAD 實測音訊的人聲起點，從 lrclib 候選中挑「第一句時間最接近」的版本（實測 Shape of You MV：同時長候選第一句從 9.86s 到 15.82s 都有，靠時長根本挑不準）。
  - ⚠️ 教訓：曾實作「VAD 起點自動平移整份時間軸」，實測把 Styx - Babe 的正確起點（37.8s）平移到 0.3s（VAD 把前奏和聲判為人聲）——**挑選是安全的（不改時間軸），平移是危險的**，已移除平移。
- **影片／音樂資料分流**：右側「📚 影片庫 / 🎵 音樂盒」、「⭐ 我的最愛 / 🎤 我的歌詞」，左側「❤️ 喜歡的頻道」也分「🎬 影片 / 🎵 音樂」兩類，各自獨立排序與刪除，互不干擾；切換輸入模式時三個側欄會自動連動切到對應分類。

## 🟢 容易（小改動，立即有感）

### 1. 聽寫模式：寬鬆比對 + 標出錯字
目前是「完全一字不差」才算對。改成忽略大小寫與標點，並把打錯／漏掉的字標紅，顯示「對了幾成」。
- **價值**：聽寫是核心功能，寬鬆比對更貼近真實學習。

### 2. 單句循環次數設定
讓使用者設定「自動重複 N 次再進下一句」。
- **價值**：適合刻意練習特定難句。

## 🟡 中等（需新增後端邏輯或資料）

### 6. 上下文翻譯（改善翻譯品質）
逐句翻譯會丟失上下文。改成翻譯時把「前一句」一併給模型當參考，但只取目前句的譯文。
- **價值**：解決代名詞、省略句的誤譯，明顯提升品質。
- **技術**：修改 `translate_one()`，prompt 帶入前句。

### 7. 單字／片語點查
點任一英文單字，彈出字典釋義（可用本地模型或離線字典）。
- **價值**：邊聽邊查，學習閉環。

### 8. 生字本與匯出
收集查過的單字，可匯出成 CSV / Anki 卡片。
- **價值**：銜接既有的記憶工具（Anki）。

### 9. 學習進度追蹤
記錄每段完成度、聽寫正確率，存在瀏覽器 localStorage 或後端。
- **價值**：成就感與回顧。

### 10. 影片庫 / 歷史清單
首頁列出處理過的影片（已快取），一鍵重開。
- **價值**：把工具變成「我的教材庫」。

### 11. 處理進度即時顯示
目前處理時前端只轉圈。改成顯示「下載中 → 轉錄中 → 翻譯中 (45/280 句)」。
- **技術**：後端改用串流 / WebSocket / 輪詢進度。
- **價值**：長時間處理時體驗大幅改善。

## 🔴 進階（較大工程或新能力）

### 12. 跟讀發音評分（Shadowing）／跟唱評分
讓使用者錄音念句子（或跟唱歌曲），比對給回饋。
- **v1（可行）**：瀏覽器錄音 + Whisper 轉錄使用者語音 + 與原句/歌詞比對 →「清晰度分數」。
- **v2（難）**：真正的音準/旋律相似度評分（音高提取、時間對齊），卡拉OK 機等級的訊號處理，等 v1 驗證有價值再投資。
- **價值**：從「聽」延伸到「說/唱」，學習維度大增。

### 13. 依難度／主題自動分級
分析詞彙難度（如 CEFR 等級），標示每段適合的程度。
- **價值**：自動推薦適合的影片片段。

### 14. AI 對話練習
針對影片內容，用本地 LLM 出問題、與使用者問答。
- **價值**：從被動聽到主動運用。

### 15. 多語言支援
不只英翻中，支援日文、西班牙文等學習（Whisper 支援多語）。
- **技術**：改用多語 Whisper 模型 + 對應翻譯。

### 16. 手機 / PWA 版
做成可安裝的 Progressive Web App，手機上也能練。
- **價值**：通勤、零碎時間都能學。

### 17. 自動產生測驗
依內容生成填空、選擇題。
- **價值**：檢驗學習成效。

## 🛠️ 穩健度與工程（持續進行）

> 你提到「測試穩健度持續進行」，這區專門記錄。

### 18. 多影片相容性測試
建立一份測試清單（不同長度、口音、領域、有無字幕的影片），定期跑過確認都能處理。

### 19. 錯誤處理與友善提示
目前錯誤多直接拋出。針對「影片無法下載」「Ollama 未啟動」「模型未下載」「whisper 未安裝」等情境給清楚的中文提示與排解建議。

### 20. yt-dlp 相容性自動因應
YouTube 改版時 yt-dlp 可能失效。可加入自動偵測並提示 `yt-dlp -U`，或嘗試多個 client（android / ios / mweb）後備。

### 21. 自動化測試
為切句、切段、翻譯解析等純邏輯函式寫單元測試，改動時不怕弄壞。

### 22. 設定面板
把模型、段落長度、聽寫秒數等做成網頁上的設定介面，不必改 `.env`。
