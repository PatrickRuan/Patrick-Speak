# 學生練習指南：用 Claude Code 從零打造這個專案

> 對象：**沒有程式背景**的學生。
> 目標：體驗「用自然語言指揮 AI 寫出一個完整可用的應用」。

這份指南會教你如何用 Claude Code Desktop，給出一個**好的起始 prompt**，讓開發過程比我們第一次更順利。我們第一次走了不少冤枉路（卡在 YouTube cookies、翻譯模型選擇、字幕抓不到），這份指南把那些教訓**預先寫進 prompt**，幫你避開。

---

## 第一步：環境準備（一次性）

1. 安裝 **Claude Code Desktop app**。
2. 安裝 **Homebrew**（macOS 套件管理工具）：到 [brew.sh](https://brew.sh) 複製指令貼到終端機。
3. 安裝 **Ollama**：到 [ollama.com](https://ollama.com) 下載。
4. 在 Claude Code 裡開一個**新資料夾**當專案目錄，例如 `MyEnglishApp`。

> 你**不需要**先懂 Python、FastAPI 或任何技術。這些 Claude 會幫你處理。你要學的是「**如何清楚描述需求**」。

---

## 第二步：選擇模型，啟用 Opus 4.8

這個專案有點難度（牽涉影音下載、語音辨識、AI 翻譯、前後端整合），**不要為了省成本用較小的模型**。

在 Claude Code 中用 `/model` 指令切換到 **Opus 4.8**（`claude-opus-4-8`）：

```
/model claude-opus-4-8
```

> 為什麼？難度高的任務，強模型能少走彎路、少出錯，整體反而更省時間。我們這次中途就是切到 Opus 4.8 才把較棘手的部分順利做完。

---

## 第三步：給出「理想的起始 prompt」

把下面這段**整段複製**貼給 Claude Code。它已經把我們踩過的坑預先講清楚了：

---

### 📋 可直接複製的 Prompt

```
請幫我建立一個「YouTube 英文聽力學習」的網頁應用。我沒有程式背景，
請你規劃技術棧、建立所有檔案、寫好前後端，並在完成後一步步教我如何啟動。

【核心流程】
輸入一個 YouTube 連結後，後端自動：
1. 下載該影片的音訊。
2. 把音訊轉成帶時間戳的英文逐句文字。
3. 以完整句子為基準，每接近 2 分鐘切成一段教材。
4. 把每一句英文翻譯成繁體中文。
然後產生一個網頁介面，每段教材有兩種模式：
- 教材播放模式：單句重複播放、上一句／下一句，並有獨立按鈕可即時切換顯示
  英文字幕、繁體中文翻譯。
- 聽寫練習模式：播放單句後自動暫停並留 10 秒供我在輸入框打字，10 秒內若沒
  操作就自動重播該句；同樣可隨時查看英文與繁中參考。

【重要技術限制，請務必遵守（這些是過來人的經驗，能幫你少走彎路）】

A. 翻譯一律使用「本地模型」，不要用任何雲端 API（不要用 Claude API / OpenAI）。
   請用我電腦上的 Ollama 來翻譯。模型我打算用 Gemma 系列（例如 gemma3:12b），
   因為它翻譯品質不錯、又不是中國來源（我對中國模型即使本地執行仍有顧慮）。
   請把模型名稱設計成可以用環境變數（.env）一行切換，方便我日後更換。

B. 字幕「不要」依賴 YouTube 既有字幕，原因有二：
   (1) 很多影片根本沒有英文字幕、或只有自動翻譯的他國語字幕；
   (2) 抓 YouTube 字幕常需要登入驗證，會要求讀取瀏覽器 cookies，這在 Mac 上
       會跳出鑰匙圈授權、形同把整個瀏覽器的 cookie 解密金鑰交出去，我不接受。
   因此請改用「本地語音轉文字」：用 whisper.cpp（brew install whisper-cpp）
   把下載的音訊直接轉成英文逐句文字。這樣不需登入、不碰 cookies、任何有英文
   語音的影片都能用。

C. 下載音訊時請預期會遇到 YouTube 的「Sign in to confirm you're not a bot」
   驗證。請「不要」用瀏覽器 cookies 解決，改用 yt-dlp 的 android 客戶端
   （--extractor-args "youtube:player_client=android"）來繞過，並用
   -f "bestaudio/best" 來適應不同影片可能不同的音訊格式。

D. whisper.cpp 需要 16kHz 單聲道 WAV 作為輸入，請用 ffmpeg 先轉檔。

E. 翻譯請「逐句」呼叫模型（一句一次），以確保中文和英文一對一對齊；不要一次
   丟整段叫模型保留編號，那樣會對齊錯位。為了速度可以用有限的並行處理。

F. 請加入快取機制：同一支影片處理過一次後，第二次開啟要能直接秒出結果。

【其他要求】
- 前端用最簡單的原生 HTML/CSS/JS 即可，不要用需要建置的框架。
- 過程中每完成一個關鍵環節，請你「自己先測試確認可行」再繼續，不要用猜的。
- 完成後請給我：如何安裝相依工具、如何下載 whisper 模型、如何啟動、如何操作。
```

## 第四步：開發中的互動技巧

- **讓 Claude 自己測試**：prompt 裡那句「自己先測試確認可行再繼續」很關鍵。能大幅減少「以為好了其實沒好」的來回。
- **遇到不懂的詞就問**：例如它說「用 ffmpeg 轉檔」，你可以直接問「ffmpeg 是什麼？為什麼需要它？」Claude 會解釋。
- **報錯就把完整錯誤訊息貼回去**：不要只說「壞掉了」，把終端機紅字整段貼給它。
- **質疑它**：如果它叫你做你覺得怪的事（例如要你輸入密碼、開權限），**停下來問清楚**。我們第一次就是在 cookies / 鑰匙圈這裡，學生若直接照做可能把瀏覽器金鑰交出去——保持懷疑是好習慣。

---

## 第五步：可以學到什麼

完成這個練習，沒有程式背景的你會親身體會到：

1. **AI 應用的組裝思維**：一個應用是把多個工具串起來（下載→轉文字→翻譯→介面），而不是一段魔法。
2. **本地 AI 的概念**：模型可以跑在自己電腦上，不必上雲、不必付費、資料不外流。
3. **「需求描述」就是新的程式語言**：你講得越清楚、越能預見問題，AI 產出越好。
4. **資安直覺**：什麼操作該警戒（交出金鑰、開權限）、什麼是安全的（本地運算）。
5. **除錯的基本節奏**：看錯誤訊息 → 回報 → 驗證 → 再前進。

---

## 第六步：做出 MVP 之後的延伸練習

上面的 prompt 會幫你做出「能用的版本」。本專案後來又一步步加了不少功能，這些都是很好的**下一輪練習題**（一次加一個，講清楚需求讓 AI 幫你加上去）：

- 長片**預設只處理前 6 分鐘 + 「全部下載」**補完（含不重翻的翻譯快取）
- **影片庫側欄**：已處理影片秒載、可排序與刪除
- **我的最愛**：收藏難句、集中複習
- **播放速度**、**亮/暗主題**切換
- 穩健性：單句翻譯失敗不崩潰、同時只處理一支

更多點子見 [ROADMAP.md](ROADMAP.md)。每加一個，都是再練一次「把需求講清楚、驗證、再前進」。

---

## 給老師的提醒

- 這份練習的價值不在「寫出 code」，而在**訓練清楚表達需求、保持資安警覺、與 AI 協作除錯**。
- 可以讓學生**故意省略**某條技術限制（例如不講 B 的 whisper），讓他們親身撞到 cookies 的牆，再體會「事先把需求講清楚」的價值——這正是我們第一次的真實經歷。
- 鼓勵學生用自己有興趣的 YouTube 影片當素材，動機更強。
